# StudyFlow 学习任务管理系统

## 1. 项目概述

StudyFlow 是一个面向学生的学习任务管理系统，支持个人任务管理与学习进度跟踪。

### 系统角色
- **个人用户**：使用该系统管理学习任务的学生

### 核心场景
- 创建任务
- 设置优先级
- 设置提醒
- 更新状态
- 查看统计

---

## 2. 功能需求

### FR1: 任务创建
系统应允许用户创建新任务，每个任务包含以下属性：
- 任务标题（必填，最长100字符）
- 任务描述（可选，最长500字符）
- 任务优先级（高/中/低）
- 截止日期（必填，不能早于创建时间）
- 任务状态（待办/进行中/已完成）

### FR2: 任务查询
系统应允许用户：
- 按优先级查询所有任务
- 查询指定日期到期的任务
- 按用户ID查询任务列表

### FR3: 任务状态管理
系统应允许用户：
- 将任务标记为已完成
- 更新任务状态（待办 ↔ 进行中 ↔ 已完成）
- 已完成任务不能再次被修改为未开始状态

### FR4: 提醒功能
系统应支持多种提醒策略：
- 到期前1天提醒
- 高优先级任务即时提醒
- 当日未完成任务汇总提醒

### FR5: 任务统计
系统应支持：
- 按用户统计任务完成情况
- 查看任务完成率

### FR6: 任务列表管理
用户应能够：
- 创建任务列表（TaskList）来组织相关任务
- 将任务添加到指定的列表中

### FR7: 进度追踪
系统应支持：
- 查看学习进度报告（ProgressReport）
- 不同统计口径的报表

### FR8: 优先级排序
系统应支持按优先级对任务进行排序显示

---

## 3. 非功能需求

### NFR1: 性能
- 任务查询响应时间不超过500ms
- 系统应能支持至少1000个并发用户

### NFR2: 可用性
- 系统可用性应达到99.9%
- 系统应提供7x24小时服务

### NFR3: 安全性
- 用户数据应加密存储
- 系统应实现基本的访问控制

### NFR4: 可维护性
- 系统代码应遵循模块化原则
- 核心模块应具有清晰的职责划分

### NFR5: 可扩展性
- 系统架构应支持功能扩展
- 应便于添加新的提醒策略

---

## 4. 验收标准

### AC1: 任务创建
- 给定有效的任务信息，系统应成功创建任务并返回任务ID
- 给定无效的截止日期（早于当前时间），系统应拒绝创建并返回错误信息

### AC2: 任务查询
- 给定用户ID，系统应返回该用户的所有任务
- 给定日期，系统应返回该日期到期的所有任务

### AC3: 状态更新
- 已完成的任务不能被修改为"待办"状态
- 系统应正确更新任务状态并返回更新后的任务

### AC4: 提醒功能
- 系统应准确触发到期前1天的提醒
- 高优先级任务应在创建后立即触发提醒

### AC5: 数据一致性
- 任务截止时间不能早于创建时间
- 高优先级任务必须具有截止日期

## 5. 核心实体与关系

### 实体 (Entity)
- **User**: 用户实体，具有唯一ID
- **Task**: 任务实体，具有唯一ID，可独立存在
- **TaskList**: 任务列表实体，用于组织任务

### 值对象 (Value Object)
- **Priority**: 优先级枚举（高/中/低）
- **TaskStatus**: 任务状态枚举（待办/进行中/已完成）
- **ReminderPolicy**: 提醒策略（不可变，包含提醒规则）

### 关键不变量
1. 任务截止时间不能早于创建时间
2. 已完成任务不能再次被修改为未开始状态
3. 高优先级任务必须具有截止日期

## 6. 系统架构

### 模块划分

```
src/
├── domain/          # 领域层：核心业务逻辑和实体
├── application/    # 应用层：用例和服务编排
├── infrastructure/  # 基础设施层：持久化、外部服务
├── interfaces/     # 接口层：API、CLI等交互入口
test/                # 测试代码
docs/                # 文档
```

### 领域层 (domain/)
- 核心实体：User, Task, TaskList
- 值对象：Priority, TaskStatus, ReminderPolicy
- 领域服务：TaskService

### 应用层 (application/)
- 用例：CreateTaskUseCase, CompleteTaskUseCase等
- DTO：数据传输对象

### 基础设施层 (infrastructure/)
- 持久化实现
- 外部服务适配器

### 接口层 (interfaces/)
- REST API控制器
- 命令行接口
