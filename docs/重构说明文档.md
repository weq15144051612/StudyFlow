# StudyFlow 项目优先级排序策略重构说明

## 1. 重构背景

### 1.1 问题描述
在 `TaskServiceImpl.listTasksByPriority()` 方法中，优先级排序规则被硬编码在方法内部：

```typescript
// 重构前的硬编码逻辑
listTasksByPriority(userId: string): Task[] {
  const tasks = this.taskRepository.findByUserId(userId);
  return tasks.sort((a, b) => {
    const priorityOrder = { [Priority.HIGH]: 0, [Priority.MEDIUM]: 1, [Priority.LOW]: 2 };
    return priorityOrder[a.priority] - priorityOrder[b.priority];
  });
}
```

### 1.2 问题分析
| 问题类型 | 具体表现 | 影响 |
|---------|---------|------|
| **违反开闭原则** | 新增排序规则需要修改核心业务代码 | 扩展性差，风险高 |
| **职责耦合** | 排序逻辑与业务逻辑混杂 | 难以独立测试和维护 |
| **可测试性差** | 排序逻辑无法单独测试 | 测试覆盖困难 |
| **硬编码** | 排序规则固定，无法动态切换 | 灵活性不足 |

### 1.3 重构目标
1. 消除硬编码，实现排序策略的可配置化
2. 引入策略模式，符合开闭原则
3. 保持原有行为不变，确保向后兼容
4. 在测试保护下完成重构

---

## 2. 设计方案

### 2.1 设计模式选择
采用 **策略模式（Strategy Pattern）**，将排序算法封装成独立的策略类，实现算法与使用算法的客户端解耦。

### 2.2 架构设计

**类图：**

```
┌─────────────────────────────────┐
│    TaskSortStrategy (接口)      │
├─────────────────────────────────┤
│ + sort(tasks: Task[]): Task[]   │
└─────────────────────────────────┘
        ▲
        │ 实现
┌───────┴─────────┐
│ PrioritySort    │
│ Strategy        │
└─────────────────┘
        ▲
        │ 依赖注入
┌─────────────────────────────────┐
│      TaskServiceImpl            │
├─────────────────────────────────┤
│ - taskRepository: Repository    │
│ - sortStrategy: Strategy        │
├─────────────────────────────────┤
│ + listTasksByPriority(): Task[] │
└─────────────────────────────────┘
```

### 2.3 核心接口与类说明

| 组件 | 类型 | 职责 |
|-----|------|------|
| `TaskSortStrategy` | 接口 | 定义排序策略契约 |
| `PrioritySortStrategy` | 类 | 实现优先级排序算法 |
| `TaskServiceImpl` | 类 | 业务服务，依赖策略接口 |

## 3. 重构步骤

### 3.1 步骤一：创建策略接口

**文件：** `src/domain/TaskSortStrategy.ts`

```typescript
import { Task } from './Task';

export interface TaskSortStrategy {
  sort(tasks: Task[]): Task[];
}
```

### 3.2 步骤二：创建策略实现类

**文件：** `src/domain/strategy/PrioritySortStrategy.ts`

```typescript
import { TaskSortStrategy } from '../TaskSortStrategy';
import { Task } from '../Task';
import { Priority } from '../Priority';

export class PrioritySortStrategy implements TaskSortStrategy {
  private priorityOrder: Record<Priority, number>;

  constructor(order?: Record<Priority, number>) {
    this.priorityOrder = order || {
      [Priority.HIGH]: 0,
      [Priority.MEDIUM]: 1,
      [Priority.LOW]: 2
    };
  }

  sort(tasks: Task[]): Task[] {
    return [...tasks].sort((a, b) => {
      return this.priorityOrder[a.priority] - this.priorityOrder[b.priority];
    });
  }
}
```

### 3.3 步骤三：修改 TaskServiceImpl

**文件：** `src/infrastructure/TaskServiceImpl.ts`

**重构前：**
```typescript
export class TaskServiceImpl implements TaskService {
  constructor(private readonly taskRepository: TaskRepository) {}

  listTasksByPriority(userId: string): Task[] {
    const tasks = this.taskRepository.findByUserId(userId);
    return tasks.sort((a, b) => {
      const priorityOrder = { [Priority.HIGH]: 0, [Priority.MEDIUM]: 1, [Priority.LOW]: 2 };
      return priorityOrder[a.priority] - priorityOrder[b.priority];
    });
  }
}
```

**重构后：**
```typescript
export class TaskServiceImpl implements TaskService {
  constructor(
    private readonly taskRepository: TaskRepository,
    private readonly sortStrategy: TaskSortStrategy = new PrioritySortStrategy()
  ) {}

  listTasksByPriority(userId: string): Task[] {
    const tasks = this.taskRepository.findByUserId(userId);
    return this.sortStrategy.sort(tasks);
  }
}
```

## 5. 重构收益分析

### 5.1 代码质量提升

| 维度 | 重构前 | 重构后 |
|-----|--------|--------|
| **开闭原则** | 违反（需修改代码扩展） | 符合（新增策略即可扩展） |
| **职责分离** | 排序逻辑与业务耦合 | 排序逻辑独立封装 |
| **可测试性** | 难以单独测试排序逻辑 | 策略类可独立单元测试 |
| **代码复用** | 排序逻辑无法复用 | 策略类可复用 |

### 5.2 扩展能力增强

**新增排序策略示例：**
```typescript
// 截止日期排序策略
class DueDateSortStrategy implements TaskSortStrategy {
  sort(tasks: Task[]): Task[] {
    return [...tasks].sort((a, b) => a.dueDate.getTime() - b.dueDate.getTime());
  }
}

// 使用方式
const service = new TaskServiceImpl(repository, new DueDateSortStrategy());
```

### 5.3 对现有功能的影响

| 影响类型 | 评估 |
|---------|------|
| **向后兼容性** | 完全兼容（默认使用原有排序逻辑） |
| **API 变更** | 无（构造函数参数为可选） |
| **性能影响** | 无显著变化 |
| **部署风险** | 低（测试覆盖完整） |

## 6. 后续扩展建议

### 6.1 策略注册机制
可引入工厂模式或依赖注入容器，实现策略的动态注册和获取：

```typescript
class StrategyFactory {
  private strategies = new Map<string, TaskSortStrategy>();

  register(name: string, strategy: TaskSortStrategy): void {
    this.strategies.set(name, strategy);
  }

  get(name: string): TaskSortStrategy | undefined {
    return this.strategies.get(name);
  }
}
```

### 6.2 配置化支持
通过配置文件动态指定排序策略：

```typescript
// config/sort-strategy.json
{
  "default": "priority",
  "strategies": {
    "priority": "PrioritySortStrategy",
    "dueDate": "DueDateSortStrategy"
  }
}
```
